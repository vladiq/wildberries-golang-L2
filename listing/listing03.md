Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
	"fmt"
	"os"
)

func Foo() error {
	var err *os.PathError = nil
	return err
}

func main() {
	err := Foo()
	fmt.Println(err)
	fmt.Println(err == nil)
}
```

Ответ:
```
Значение интерфейса равно nil тогда и только тогда, когда его значение и тип равны nil.
Если в примере использовать fmt.Printf("%T %v\n", err, err), то мы получим: *fs.PathError <nil>
То есть, значение nil у интерфейса err имеет тип, и nil без типа не равен nil с типом.

Устройство интерфейсов:
type iface struct { // 16 байт на 64-битной архитектуре, по 8 байт на каждое поле
    tab *itab //  указатель на interface table — структуру, которая хранит данные о типе и списке методов, которые используются для удовлетворения интерфейса
    data unsafe.Pointer // указывается на фактическую переменную с конкретным типом
}

Так как для удовлетворения пустого интерфейса не нужно реализовывать никакие методы, то любой тип удовлетворяет пустому интерфейсу.
В пустом интерфейсе itable не хранится.

Структура пустого интерфейса:
type eface struct { // также 16 байт на 64-битной архитектуре
    _type *_type // информация о типе, на который указывает data
    data  unsafe.Pointer
}


```
